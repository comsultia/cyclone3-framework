NAME

Google::Ads::Common::OAuth2Handler

Back to Top

DESCRIPTION

A partial implementation of the Google::Ads::Common::OAuthHandlerInterface manpage that defines most of the logic required to use OAuth2 against Google APIs.

It is meant to be specialized and its _scope method be properly implemented.

Back to Top

ATTRIBUTES

Each of these attributes can be set via Google::Ads::Common::OAuth2Handler->new().

Alternatively, there is a get_ and set_ method associated with each attribute for retrieving or setting them dynamically.

api_client

A reference to the API client used to send requests.

client_id

OAuth2 client id obtained from the Google APIs Console.

client_secret

OAuth2 client secret obtained from the Google APIs Console.

access_type

OAuth2 access type to be requested when following the authorization flow. It defaults to offline but it can be set to online.

approval_prompt

OAuth2 approval_prompt to be used when following the authorization flow. It defaults to auto but it can be set to always - to force the user to always authorize.

redirect_uri

Redirect URI as set for you in the Google APIs console, to which the authorization flow will callback with the verification code. Defaults to urn:ietf:wg:oauth:2.0:oob for the installed applications flow.

access_token

Stores an OAuth2 access token after the authorization flow is followed or for you to manually set it in case you had it previously stored. If this is manually set this handler will verify its validity before preparing a request.

refresh_token

Stores an OAuth2 refresh token in case of an offline the access_type manpage is requested. It is automatically used by the handler to request new access tokens i.e. when they expire or found invalid.

initialize

Initializes the handler with properties such as the consumer_key and consumer_secret to use for generating authorization requests.

Parameters

  • A required api_client with a reference to the API client object handling the requests against the API.

  • A hash reference with the following keys: { # Refer to the documentation of the the client_id manpage property. oAuth2ClientId => "consumer key", # Refer to the documentation of the the client_secret manpage property. oAuth2ClientSecret => "consumer secret", # Refer to the documentation of the the access_type manpage property. oAuth2AccessType => "name", # Refer to the documentation of the the approval_prompt manpage property. oAuth2ApprovalPrompt => "token", # Refer to the documentation of the the access_token manpage property. oAuth2AccessToken => "secret", # Refer to the documentation of the the refresh_token manpage property. oAuth2RefreshToken => "secret", # Refer to the documentation of the the redirect_uri manpage property. oAuth2RedirectUri => "secret", }
